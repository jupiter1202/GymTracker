Create PlanDetailScreen.kt. This is the most complex screen — take care with gesture layering.

**Structure:**
```kotlin
@Composable
fun PlanDetailScreen(planId: Long, onNavigateBack: () -> Unit)
```

1. Obtain `WorkoutPlanViewModel` via `koinViewModel()`. Call `LaunchedEffect(planId) { viewModel.setActivePlan(planId) }` to load exercises.
2. Collect `planExercises` and `plans` StateFlows. Derive current plan: `val plan = plans.find { it.id == planId }`.
3. Local state: `var localExercises by remember(planExercises) { mutableStateOf(planExercises) }` — maintains in-flight drag order.
4. Set up reorderable: `rememberReorderableLazyListState(lazyListState) { from, to -> ... }`.

**TopAppBar:**
- Title: plan name
- Navigation icon: back arrow (Icons.AutoMirrored.Filled.ArrowBack)
- Action: IconButton with Edit icon → opens rename sheet

**LazyColumn body:**
- Each exercise row is a `ReorderableItem` wrapping a `SwipeToDismissBox` wrapping `ExerciseRow`
- `ExerciseRow` composable: Row with drag handle IconButton (draggableHandle), exercise name + muscle group chip, "3 × 8" reps text, clickable → opens target edit sheet
- SwipeToDismissBox background: red Container with Delete icon (Icons.Default.Delete), only for EndToStart
- On drop: `viewModel.reorderExercises(localExercises.map { it.planExercise })`
- On swipe: `viewModel.removeExercise(item.planExercise)`

**"+ Add exercise" item:**
- Regular ListItem at the bottom of the LazyColumn (after all exercise items)
- Tapping opens the exercise picker sheet

**Exercise picker bottom sheet:**
- Full-height ModalBottomSheet
- Obtain `ExerciseViewModel` via `koinViewModel()` — separate instance for search state
- Same search bar + muscle group filter chips as ExercisesScreen (import MUSCLE_GROUPS from core/constants)
- LazyColumn of exercise rows; tapping an exercise row dismisses picker and opens target input sheet

**Target input sheet (for both add and edit):**
- `var targetSets by remember { mutableIntStateOf(3) }`
- `var targetReps by remember { mutableStateOf("8") }`
- Pre-fill from existing planExercise if editing
- Number field (OutlinedTextField with keyboardOptions = KeyboardOptions(keyboardType = KeyboardType.Number)) for sets
- Text field for reps (supports "5", "8-12", "AMRAP")
- Cancel / Add to plan (or Save) buttons
- On save (add): `viewModel.addExercise(planId, exerciseId, sets, reps)`
- On save (edit): `viewModel.updateExerciseTargets(planExercise, sets, reps)`

**Rename plan bottom sheet:**
- Same CreatePlanSheet pattern with pre-filled name + description
- On save: `viewModel.updatePlan(plan.copy(name = newName, description = newDesc))`

**Sheet state management:**
- `var showExercisePicker by remember { mutableStateOf(false) }`
- `var pendingExercise by remember { mutableStateOf<Exercise?>(null) }` — set when exercise selected from picker, drives target input sheet
- `var editingPlanExercise by remember { mutableStateOf<PlanExerciseWithExercise?>(null) }` — set when tapping existing row
- `var showRenameSheet by remember { mutableStateOf(false) }`

Follow RESEARCH.md anti-patterns:
- Do NOT place SwipeToDismissBox outside ReorderableItem — wrap order must be ReorderableItem > SwipeToDismissBox > row content
- Use stable Long id as key for both items() and ReorderableItem
- On every drop, reindex the full list 0..N-1 via reorderExercises; DB gaps don't matter

All imports: sh.calvin.reorderable (ReorderableItem, rememberReorderableLazyListState, draggableHandle), Material3 (SwipeToDismissBox, SwipeToDismissBoxValue, rememberSwipeToDismissBoxState, ExperimentalMaterial3Api, TopAppBar, Scaffold, ModalBottomSheet, rememberModalBottomSheetState, ListItem, ElevatedCard, FilterChip, OutlinedTextField, KeyboardOptions, KeyboardType), Icons (ArrowBack via AutoMirrored, DragHandle, Delete, Edit, Add), LazyColumn (rememberLazyListState, items), lifecycle (LaunchedEffect, collectAsStateWithLifecycle), koin (koinViewModel).
