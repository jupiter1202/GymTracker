## Decisions Made

- WorkoutSession.planId is nullable with SET_NULL on parent delete — preserves historical session data when a workout plan is deleted
- Composite index on WorkoutSet [session_id, exercise_id] — covers the primary query pattern "show all sets for exercise X in session Y"
- All column names use explicit @ColumnInfo snake_case — prevents Room from using Kotlin camelCase column names which could cause issues in raw SQL queries
