<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human smoke test — 8 workout logging scenarios</name>
  <what-built>
    Complete workout logging feature:
    - ActiveWorkoutScreen (LazyColumn with exercise sections, inline set input, sticky rest timer banner, elapsed time)
    - WorkoutSummaryScreen (totals: name, duration, exercises, sets, volume)
    - Dashboard quick-start card (recent plan + start empty workout)
    - Plans screen Start button per plan card
    - Crash recovery (app close during workout → reopen → resumes session)
    - Post-hoc session entry with date picker
    - SettingsScreen rest timer duration setting
    - Rest timer with vibration + sound alert
  </what-built>
  <how-to-verify>
    Install the app on a device or emulator. Run each scenario in order.

    **Scenario 1: Start workout from Plans tab (LOG-01, D-06)**
    1. Navigate to Plans tab
    2. Tap "Start" on any plan card
    3. EXPECTED: Active workout screen opens showing the plan's exercises in sections
    4. EXPECTED: Each exercise has an inline pending-set row pre-filled with weight and reps

    **Scenario 2: Log a set and rest timer (LOG-01, LOG-03, D-03, D-10, D-11)**
    1. In the active workout screen, edit the weight and reps fields for the first exercise
    2. Tap the checkmark (✓) button
    3. EXPECTED: Set is logged and appears as a row in the exercise section
    4. EXPECTED: Rest timer banner appears immediately below the top bar showing a countdown (e.g. "Rest · 1:30")
    5. EXPECTED: Banner stays visible as you scroll down
    6. Tap "Skip" on the banner
    7. EXPECTED: Banner disappears

    **Scenario 3: Elapsed time (LOG-05, D-14)**
    1. While in active workout screen, observe the top app bar
    2. EXPECTED: Timer shows elapsed time in format "0:00:xx" and increments every second

    **Scenario 4: Previous performance (LOG-04, D-02)**
    1. Complete a full workout (at least one set per exercise) and save it
    2. Start a new workout with the same plan
    3. EXPECTED: Each exercise section header shows "Last: Nx8 @ XX kg" (or lbs) from the previous session

    **Scenario 5: Add and remove exercises mid-workout (D-04, D-05, D-09)**
    1. In active workout, tap "+ Add exercise" at the bottom
    2. EXPECTED: Exercise picker bottom sheet opens with search and filter
    3. Pick an exercise
    4. EXPECTED: New exercise section added at the bottom of the list
    5. Tap the ✕ button on the new exercise section
    6. EXPECTED: Section is removed

    **Scenario 6: Finish workout and summary (D-15, D-16, D-17)**
    1. In active workout, log at least 2 sets for at least 1 exercise
    2. Tap "Finish" in the top bar
    3. EXPECTED: Workout Summary screen shows workout name, duration, exercise count, set count, and volume in your unit
    4. Tap "Done"
    5. EXPECTED: Navigates to History tab

    **Scenario 7: Crash recovery (D-19)**
    1. Start a new workout from Dashboard
    2. Log at least one set
    3. Force-stop the app (Settings → Apps → GymTracker → Force Stop, or swipe away from recents)
    4. Reopen the app
    5. EXPECTED: App navigates directly to the active workout screen (no prompt, no extra taps)
    6. EXPECTED: The previously logged set is still visible

    **Scenario 8: Rest timer alert and settings (D-12, D-13)**
    1. Go to Settings tab
    2. EXPECTED: A "Rest Timer" section is visible with a text field showing "90"
    3. Change the value to "30"
    4. Go back, start a workout, log a set
    5. EXPECTED: Rest timer starts at 30 seconds (not 90)
    6. Wait for timer to reach 0 (or let it run down on a physical device)
    7. EXPECTED: Vibration occurs at timer completion (physical device only)
  </how-to-verify>
  <resume-signal>Type "approved" if all 8 scenarios pass, or describe which scenarios failed and what you observed.</resume-signal>
