<objective>
Fix the root cause of the Flow subscription race condition: ActiveWorkoutScreen unconditionally calls resumeSession(), which overwrites fresh session exercises with empty data from the initial Flow emission.

**Root Cause:** 
1. PlansScreen.startSessionAndGetId() populates _exerciseSections with exercises ✓
2. ActiveWorkoutScreen LaunchedEffect calls resumeSession(sessionId) **unconditionally** ✗
3. resumeSession() loads from database via Flow<List<WorkoutSet>>
4. Flow's first emission on fresh session: empty list (no sets logged yet)
5. Line 245 sets _exerciseSections.value = emptyList() ← **exercises cleared**

**Solution:** Add null check to activeSession before calling resumeSession — only load from DB if this is crash recovery (activeSession is null), not a fresh start.

Purpose: Prevent resumeSession's first Flow emission from clearing exercises on fresh starts while maintaining crash recovery functionality
Output: Fixed ActiveWorkoutScreen + WorkoutLoggingViewModel ready for verification
</objective>
